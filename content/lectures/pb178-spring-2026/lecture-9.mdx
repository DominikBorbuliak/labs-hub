# Asynchronous Programming

## What Is Asynchronous Programming?

**Parallel programming** is about doing multiple things **at the same time** — using multiple threads to speed up CPU-bound work.

**Asynchronous programming** is about **not blocking** while waiting for something to finish — freeing up the current thread to do other work while an I/O operation completes (e.g., reading a file, calling an API, querying a database).

|                 | Parallel                                | Async                                  |
| --------------- | --------------------------------------- | -------------------------------------- |
| **Goal**        | Speed up CPU-bound work                 | Keep the app responsive during I/O     |
| **Mechanism**   | Multiple threads running simultaneously | Release the thread while waiting       |
| **Typical use** | Data processing, calculations           | Web requests, file I/O, database calls |

> [!IMPORTANT]
> Async does **not** mean "runs on another thread." It means "I can do something else while this operation completes." The operation might not use a thread at all (e.g., waiting for a network response).

## Async and Await

In C#, asynchronous programming is built around the `async` and `await` keywords.

```csharp
using System;
using System.Threading.Tasks;

// An async method returns Task (or Task<T> if it returns a value)
async Task<string> FetchDataAsync()
{
    Console.WriteLine("Starting to fetch data...");
    await Task.Delay(2000); // Simulates waiting for an I/O operation (non-blocking)
    Console.WriteLine("Data fetched!");
    return "Server response";
}

// Calling the async method
string result = await FetchDataAsync();
Console.WriteLine($"Received: {result}");
```

Key rules:

- Mark the method with `async`
- Return `Task` (no result) or `Task<T>` (with result)
- Use `await` to asynchronously wait for the result — but you don't have to `await` immediately. You can store the `Task` and `await` it later, which lets you start multiple operations before waiting for any of them
- Name async methods with the **`Async` suffix** (e.g., `FetchDataAsync`, `LoadUserAsync`) — this is a widely followed .NET convention that makes it clear the method returns a `Task` and should be awaited
- `async void` should **only** be used for event handlers — never for regular methods

```csharp
// ❌ async void — three serious problems:
// 1. Caller can't await it — no way to know when it finishes
// 2. Exceptions crash the process — they can't be caught by the caller
// 3. Makes testing impossible — no Task to assert on
async void LoadData()
{
    var data = await FetchDataAsync(); // If this throws, the entire app crashes!
    Console.WriteLine(data);
}

// ✅ async Task — caller can await, catch exceptions, and test
async Task LoadDataAsync()
{
    var data = await FetchDataAsync();
    Console.WriteLine(data);
}
```

```csharp
// Why async void is dangerous — exception behavior

async void DangerousAsync()
{
    throw new Exception("Boom!"); // This will CRASH the entire application!
}

async Task SafeAsync()
{
    throw new Exception("Boom!"); // This can be caught by the caller with try/catch
}

async Task ShowDangerousAsync()
{
    try
    {
        DangerousAsync(); // Exception escapes — cannot be caught here!
    }
    catch (Exception) { /* Never reached */ }
}

async Task ShowSafeAsync()
{
    try
    {
        await SafeAsync(); // Exception is properly caught
    }
    catch (Exception) { /* This works */ }
}
```

## Sequential vs Concurrent Async

Just because you use `async`/`await` doesn't mean things run concurrently. If you `await` each call one by one, they run **sequentially**:

```csharp
// Sequential — each operation waits for the previous one to finish
async Task SequentialAsync()
{
    string user = await FetchUserAsync();       // ~1 second
    string orders = await FetchOrdersAsync();   // ~1 second
    string reviews = await FetchReviewsAsync();  // ~1 second
    // Total: ~3 seconds
}
```

To run independent operations **concurrently**, start them all first, then await them together:

```csharp
// Concurrent — all operations start at the same time
async Task ConcurrentAsync()
{
    Task<string> userTask = FetchUserAsync();       // Started
    Task<string> ordersTask = FetchOrdersAsync();   // Started
    Task<string> reviewsTask = FetchReviewsAsync(); // Started

    // Now await them all — they're already running
    string user = await userTask;
    string orders = await ordersTask;
    string reviews = await reviewsTask;
    // Total: ~1 second (the longest one)
}
```

## Task.WhenAll and Task.WhenAny

`Task.WhenAll` waits for **all** tasks to complete. Use it when you need every result before proceeding.

```csharp
async Task LoadDashboardAsync()
{
    Task<User> userTask = GetUserAsync();
    Task<List<Order>> ordersTask = GetOrdersAsync();
    Task<Stats> statsTask = GetStatsAsync();

    // Wait for all three to complete
    await Task.WhenAll(userTask, ordersTask, statsTask);

    // All results are now available
    Console.WriteLine($"User: {userTask.Result}");
    Console.WriteLine($"Orders: {ordersTask.Result.Count}");
    Console.WriteLine($"Stats: {statsTask.Result}");
}
```

`Task.WhenAny` returns as soon as **any one** task completes. Use it when you only need the fastest result or want to implement timeouts.

```csharp
async Task<string> FetchWithTimeoutAsync()
{
    Task<string> fetchTask = FetchDataAsync();
    Task timeoutTask = Task.Delay(5000);

    Task completedTask = await Task.WhenAny(fetchTask, timeoutTask);

    if (completedTask == fetchTask)
    {
        return await fetchTask; // Data arrived in time
    }
    else
    {
        throw new TimeoutException("Fetch took too long!");
    }
}
```

## Best Practices and Common Pitfalls

```csharp
// ❌ Blocking on async code — can cause deadlocks, especially in UI apps
string result = FetchDataAsync().Result; // BLOCKS the thread!
FetchDataAsync().Wait();                 // BLOCKS the thread!

// ✅ Await properly
string result = await FetchDataAsync();
```

```csharp
// ❌ Unnecessary async/await — just wrapping a return value
async Task<int> GetValueAsync()
{
    return await Task.FromResult(42); // Pointless overhead
}

// ✅ Return the task directly when there's no additional logic
Task<int> GetValueAsync()
{
    return Task.FromResult(42);
}
```

> [!TIP]
> **Async all the way** — once you start using `async`/`await`, use it consistently through the entire call chain. Mixing sync and async code is the #1 source of deadlocks.

## How It Works

Every kitchen action is represented by calling this helper method:

```csharp
static async Task DoActionAsync(string actionName)
{
    Console.WriteLine($"[{DateTime.Now:HH:mm:ss}] Started: {actionName}");
    await Task.Delay(Random.Shared.Next(1000, 5001));
    Console.WriteLine($"[{DateTime.Now:HH:mm:ss}] Finished: {actionName}");
}

// Usage examples:
await DoActionAsync("Chopping tomatoes");
await DoActionAsync("Grilling patty");
await DoActionAsync("Baking pizza in oven");
await DoActionAsync("Slicing mushrooms");
```

You can also create **subtask methods** that group related actions together. For example, a `PrepareToppingsAsync` method that internally chops tomatoes, slices mushrooms, grates cheese, etc.:

```csharp
static async Task PrepareToppingsAsync()
{
    // Each of these is a separate action inside the subtask
    await DoActionAsync("Chopping tomatoes");
    await DoActionAsync("Slicing mushrooms");
    await DoActionAsync("Grating cheese");
}
```

Think about which actions inside a subtask can run at the same time and which must wait for each other.

```csharp
using System;
using System.Threading.Tasks;

// Helper method for simulating kitchen actions
static async Task DoActionAsync(string actionName)
{
    Console.WriteLine($"[{DateTime.Now:HH:mm:ss}] Started: {actionName}");
    await Task.Delay(Random.Shared.Next(1000, 5001));
    Console.WriteLine($"[{DateTime.Now:HH:mm:ss}] Finished: {actionName}");
}

// Subtask: Prepare filling (multiple independent subtasks)
static async Task PrepareFillingAsync()
{
    Console.WriteLine("--- Preparing filling ---");

    Task dicePeppers = DoActionAsync("Dicing peppers");
    Task sliceOnions = DoActionAsync("Slicing onions");
    Task shreddingCheese = DoActionAsync("Shredding cheese");
    Task cookChicken = DoActionAsync("Grilling and slicing chicken");

    await Task.WhenAll(dicePeppers, sliceOnions, shreddingCheese, cookChicken);

    Console.WriteLine("--- All filling ready ---");
}

// Subtask: Prepare salsa
static async Task PrepareSalsaAsync()
{
    Task chopTomatoes = DoActionAsync("Chopping tomatoes");
    Task minceCilantro = DoActionAsync("Mincing cilantro and jalapeño");

    await Task.WhenAll(chopTomatoes, minceCilantro);

    await DoActionAsync("Mixing salsa ingredients");
}

// Prepare a single quesadilla
static async Task PrepareQuesadillaAsync(int orderNumber)
{
    Console.WriteLine($"\n=== Quesadilla #{orderNumber} — Starting preparation ===");

    // Filling and salsa can be prepared at the same time
    Task fillingTask = PrepareFillingAsync();
    Task salsaTask = PrepareSalsaAsync();
    Task heatGriddle = DoActionAsync("Heating the griddle");

    await Task.WhenAll(fillingTask, salsaTask, heatGriddle);

    // Assembly must wait for filling
    await DoActionAsync("Laying tortilla and adding filling");
    await DoActionAsync("Folding and pressing quesadilla");

    // Cooking must wait for assembly and griddle
    await DoActionAsync("Cooking quesadilla on griddle — first side");
    await DoActionAsync("Flipping and cooking second side");

    // Final step
    await DoActionAsync("Cutting into wedges and plating with salsa");

    Console.WriteLine($"=== Quesadilla #{orderNumber} — Ready to serve! ===\n");
}

// Handle multiple orders concurrently
Console.WriteLine("Restaurant is open! Taking orders...\n");

Task order1 = PrepareQuesadillaAsync(1);
Task order2 = PrepareQuesadillaAsync(2);
Task order3 = PrepareQuesadillaAsync(3);

await Task.WhenAll(order1, order2, order3);

Console.WriteLine("All orders completed!");
```
